## Core Workflows

### Health Check
```bash
set -euo pipefail
cd /home/garet/SFS/[REPO_NAME]
curl -s http://localhost:5000/health | grep '"ok":true'
```
Standard: `GET /health → {"ok":true}` — all SFS apps must expose this.

### CI/CD Deploy
```bash
git push origin main           # triggers GitHub Actions
gh run list --limit 5          # check status
gh run rerun [RUN_ID]          # retry failed run
```
CI source of truth: `smartflow-systems/SmartFlowSite/.github/workflows/ci.yml`

Required secrets: `SFS_PAT` (required), `REPLIT_TOKEN`, `SFS_SYNC_URL`

### Scaffold CI for New Repo
```bash
set -euo pipefail
mkdir -p .github/workflows
curl -s https://raw.githubusercontent.com/smartflow-systems/SmartFlowSite/main/.github/workflows/ci.yml \
  -o .github/workflows/ci.yml
git add .github/workflows/ci.yml
git commit -m "ci: add SFS standard CI workflow"
git push origin main
```

### Secrets Management
```bash
gh secret set SFS_PAT --body "$SFS_PAT"
gh secret set REPLIT_TOKEN --body "$REPLIT_TOKEN"
gh secret set SFS_SYNC_URL --body "$SFS_SYNC_URL"
# Org-level:
gh secret set SFS_PAT --org smartflow-systems --body "$SFS_PAT"
```

## Troubleshooting

| Symptom | Root Cause | Fix |
|---------|-----------|-----|
| CI failing | Missing secret | `gh secret set [NAME]` |
| Health check fails | Wrong port | Check `PORT` env var |
| Build fails | Missing deps | `npm ci` or `pip install -r requirements.txt` |
| Deploy times out | Replit cold start | Increase timeout in workflow |
| SFS_PAT permission denied | Expired token | Rotate at GitHub → Settings → Tokens |
| Webhook signature fail | Wrong `whsec_` | `stripe listen` to get correct secret |
